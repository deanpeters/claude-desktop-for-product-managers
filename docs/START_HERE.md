# Start Here

Contributor reading path. (Learners start at `course-materials/README.md` instead.)

Before changing anything, read in order:

1. `README.md`
2. `CONSTITUTION.md`
3. `PLAN.md`
4. `docs/course-architecture.md`

Then pick your lane:

- **Course design** — work in `course-materials/`. This is the product.
- **Repo framing** — root docs and `docs/`.
- **Reference layer** — `website/` (Astro + Starlight). Summarize lessons, never fork them; `course-materials/` stays canonical.

## Rules of the road

- Keep it clean-room. Nothing imported, nothing paraphrased — every word written for this course.
- One complete lesson beats seven vague ones.
- No terminal required, ever, for the learner path.
- Never promise automation that isn't wired up.
- Realistic PM artifacts only — things a working PM would recognize on sight.
